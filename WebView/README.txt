Введение
-------------------------------------------------------------------------------
Эти две папки являются частью проекта, созданного с помощью Node.js с 
использованием create-react-app.
///////////////////////////////////////////////////////////////////////////////

Что это за папки?
-------------------------------------------------------------------------------
В папке src/ находятся исходныее файлы, напсанные мной, в которых реализована 
логика работы сайта. В build/ лежат файлы, составленные системой по файлам из 
папки src/, предназначенные для деплоя.

package.json показывает версии используемых в проекте утилит,
///////////////////////////////////////////////////////////////////////////////

Запуск проекта
-------------------------------------------------------------------------------
Для того, чтобы запустить проект на своей машине, требуется сначала установить 
Node.js и установить через PowerShell/терминал create-react-app, затем создать
проект (обратитесь к инструкции в интернете, ссылка ниже) и заменить файлы в 
папке src/ после создания проекта. 
Скопируйте в проект папку build, затем запустите полученное с помощью команды
npm run build. Затем нужно перейти по адресу, который будет указан в 
PowerShell/терминале.

Ссылка на инструкцию:
https://github.com/facebookincubator/create-react-app
///////////////////////////////////////////////////////////////////////////////

Навигация на сайте
-------------------------------------------------------------------------------
Если вы все сделали правильно, то по адресу, по которому вы перешли, вы 
увидите страницу /#/ с надписью Hi. Это временная заглушка, здесь далее 
будет страница авторизации.

Для того, чтобы перейти на страницу пользователя, введите в адресной строке 
.../#/user/0
Пока что создан только один пользователь, поэтому других страниц пользователя 
нет.

Для того, чтобы перейти на страницу книги, введите в адресной строке
.../#/book/0 | .../#/book/1 | .../#/book/2

Для того, чтобы перейти на страницу автора, введите в адресной строке
.../#/author/0 | .../#/author/1 | .../#/author/2

Для того, чтобы увидеть список книг пользователя, введите в адресной строке
.../#/user/0/list
Отдельно хочу заметить, что со страницы пользователя можно попасть на станицу 
со списком его книг, а оттуда - на страницу книги. Данная возможность 
реализована в виде заглушки. 
Для страницы автора данная возможность временно не предусмотрена, поскольку 
после связи между страницами будут определяться с помощью запросов.
